扫描所有非排除目录的 `.js/.jsx/.mjs/.cjs` 源文件，按固定规则机械归类，识别可 TS 化候选并输出报告。**已脚本化**（同 git state 下两次执行 byte-byte 一致）。

## 调用方式

```bash
mkdir -p reports/$(date +%Y-%m-%d)
node .claude/skills/audit/ts-candidates/scripts/audit.mjs \
  > reports/$(date +%Y-%m-%d)/byd-audit-ts-candidates-$(date +%Y-%m-%d).md
```

## 输出位置

`reports/YYYY-MM-DD/byd-audit-ts-candidates-YYYY-MM-DD.md`（reports/ 已 gitignore）

## Cross-check 锚点

脚本内置 1 个 sanity check：主统计的候选文件总数 vs `system find` 独立统计。一致才能交付。

下方"判定规则 / 执行步骤 / 已知陷阱"为脚本逻辑的详细说明（脚本内 `classify()` 实现的就是这套规则）。

## 设计目标

**口径稳定**：本 skill 不依赖经验判断，所有归类基于「路径模式 + 内容首行 + 文件大小」的机械规则。同一个仓库快照，多次运行结果必须一致。

## 扫描范围

### 包含

`*.js`、`*.jsx`、`*.mjs`、`*.cjs`

### 强制排除目录（执行 find 时 -prune）

```
node_modules .next out out-rtl dist build .git .turbo coverage public reports
```

并通过 `-path '*/apps-icons/*' -prune` 排除 apps-icons 子树（图标包，1000+ 编译产物）。

## 判定规则（按优先级从上到下，命中即停）

### 规则 0：内容预扫描

对每个文件读取：

- `lines = wc -l`
- `first = head -1`
- `size = stat -f%z`（macOS）

### 规则 1：自动归类「不应转」

> 实际优先级：**credential 最优先**（陷阱 #7），其次 i18n-data（陷阱 #1），其余按下表自上而下。

| 触发条件                                                                                                   | 标签            | 原因                                                                 |
| ---------------------------------------------------------------------------------------------------------- | --------------- | -------------------------------------------------------------------- |
| 内容含 `client_secret`、`access_token`、`refresh_token`、`private_key`                                     | credential      | **应当从仓库清除而非转 TS**（必须最优先，避免被 node-script 等掩盖） |
| 路径匹配 `*/i18n/src/meta/*/*.js` 或 `*/meta/*/data/*.js`                                                  | i18n-data       | 纯数据，CLAUDE.md 注明动态执行                                       |
| 扩展名 `.cjs`                                                                                              | config          | CommonJS 强制                                                        |
| 扩展名 `.mjs` 且文件名匹配 `*.config.mjs`                                                                  | config          | 配置文件                                                             |
| 文件名匹配 `*.config.js`                                                                                   | config          | 配置文件（next/postcss/pm2/analyze 等）                              |
| 文件名匹配 `analyze.js`                                                                                    | config          | bundle 分析脚本                                                      |
| 文件名匹配 `.commitlintrc.js`、`.lintstagedrc.js`、`.eslintrc.js`                                          | config          | lint 工具配置                                                        |
| 路径包含 `/plugin/babel-plugin-`                                                                           | babel-plugin    | Babel 同步加载，必须 CommonJS                                        |
| 路径包含 `apps-base-kline/`                                                                                | kline-pkg       | 独立发布 npm 包，自有构建链                                          |
| 路径包含 `/scripts/` 或文件路径以 `scripts/` 开头                                                          | node-script     | 项目惯例：开发期 Node 脚本不转 TS                                    |
| 文件名匹配 `server.js` 且位于 `apps/*/server.js`                                                           | next-server     | 项目无 tsx/ts-node 依赖（仅 apps-icons 子包有）                      |
| 首行命中 generated 标记（不区分大小写）：`code generated`、`@generated`、`auto-?generated`、`generated by` | generated       | 生成产物                                                             |
| `lines == 1 && size > 5000`                                                                                | minified-vendor | 压缩后的第三方代码                                                   |
| 文件名 ∈ `{gt4.js, charting_library.esm.js}`                                                               | vendor-sdk      | 已知第三方 SDK 白名单                                                |

### 规则 2：自动归类「不建议转（收益过低）」

| 触发条件                                                                              | 标签            | 原因                            |
| ------------------------------------------------------------------------------------- | --------------- | ------------------------------- |
| `lines <= 6` 且 grep 仅匹配 `^export\s*\{.*\}\s*from`                                 | barrel-only     | 纯重导出，无类型可加            |
| 文件主体仅 `export const X = \`...\`` 字符串字面量（无 import 复杂逻辑 / 无函数调用） | string-template | 字符串自动推断为 string，零收益 |
| 文件主体仅 `module.exports = {大对象字面量}`，无函数/方法                             | data-only       | 数据对象，无类型可加            |
| 文件主体仅 `const X = {...}` + 单个 export，无方法                                    | data-only       | 同上                            |

### 规则 3：自动归类「建议转」

| 触发条件                                                        | 标签          | 优先级                                                                        |
| --------------------------------------------------------------- | ------------- | ----------------------------------------------------------------------------- |
| 扩展名 `.jsx` 且同目录有 `.tsx` 兄弟文件                        | jsx-mixed     | 高（一致性）                                                                  |
| 路径包含 `chart/k-chart/lib/` 且不在 `vendor/` 子路径下         | chart-adapter | 高（项目自写适配器；真 vendor 已被 `vendor-sdk` 白名单提前排除，详见陷阱 #5） |
| 含 `class\s+\w+` 或 `function\s+\w+` 多次（≥3）且未命中规则 1/2 | logic-code    | 中                                                                            |

### 规则 4：默认归类

未命中以上任何规则的文件 → 标签 `gray-area`，需人工 review。

## 执行步骤

### 阶段 1：枚举与采集

```bash
find . \( -name node_modules -o -name .next -o -name out -o -name out-rtl \
       -o -name dist -o -name build -o -name .git -o -name .turbo \
       -o -name coverage -o -name public -o -name reports \
       -o -path '*/apps-icons/*' \) -prune \
       -o -type f \( -name '*.js' -o -name '*.jsx' -o -name '*.mjs' -o -name '*.cjs' \) -print \
  | sort > /tmp/ts-candidates-list.txt

# 采集每个文件元数据：lines | path | first_line
while IFS= read -r f; do
  lines=$(wc -l <"$f" | tr -d ' ')
  size=$(stat -f%z "$f" 2>/dev/null || stat -c%s "$f")
  first=$(head -1 "$f" 2>/dev/null | head -c 300)
  printf "%s\t%s\t%s\t%s\n" "$lines" "$size" "$f" "$first"
done < /tmp/ts-candidates-list.txt > /tmp/ts-candidates-meta.txt
```

### 阶段 2：机械归类

按规则 1→2→3→4 顺序，对每个文件标注标签。实现方式：node 一次性脚本（`/tmp/ts-classify.js`）读取 meta，逐行匹配规则，输出 `tag\tpath\treason` 到 `/tmp/ts-classified.txt`。

**关键判定细节**：

- 规则 1 的「barrel-only」检测：`lines <= 6` 时全文 grep `'^export\s*\{.*\}\s*from'`，纯 export-from 行 + 空行总和 == lines
- 规则 1 的「string-template」检测：`grep -c '^export const \w\+ = \`'`≥ 1 且整文件没有`function `、`class `、`=>`、`if `、`for `、`while ` 关键字
- 规则 1 的「data-only」检测：首行 ∈ {`module.exports = {`, `module.exports ={`, `const \w+ = {`, `export const \w+ = {`}，且整文件无 `function`、`class`、`=>`、`return `（仅有数据字面量）
- 规则 1 的 credential 检测：`grep -lE '(client_secret|access_token|refresh_token|private_key)\s*[:=]'` 命中即归类（注意：`token.js`、`credentials.js` 已用文件名豁免，避免漏判）
- 规则 3 的 chart-adapter：路径必须严格匹配 `*/chart/k-chart/lib/*` 且**不在**子路径含 `vendor/` 的位置

### 阶段 3：报告生成

**报告只输出可改的清单**，给团队照着改用。所有「不应转 / 不建议转 / 灰色地带 / 含凭证」都不进报告——这些是 skill 内部分类，对执行 TS 化任务的团队无用。

#### chart 适配器豁免清单（已知收益低，不进可改清单）

以下文件路径虽命中 `chart-adapter`，但实际收益低，**不输出到报告**：

- `packages/apps-kit/components/chart/k-chart/lib/trading-view/library/save_load_adapter.js`（1717 行 95% 是 TradingView 配置数据，末尾几行才是方法）
- `packages/apps-kit/components/chart/k-chart/lib/trading-view/library/time-zome.js`（35 行仅 1 个小工具函数）

豁免后归类为 `chart-low-value`，与 barrel-only / string-template / data-only 一并不进可转清单。

#### 报告格式（极简）

写入 `reports/YYYY-MM-DD/byd-web-report-ts-candidates-YYYY-MM-DD.md`：

```markdown

> 生成时间：YYYY-MM-DD HH:mm
> 共 N 个文件待转 TS

- `path1.jsx`（N 行）
```

无概览表、无统计、无优先级建议、无安全提醒——这些不属于「可改清单」。如团队需要规则解释，再看 SKILL.md。

## 已知陷阱（首版踩坑沉淀）

### 1. i18n-data 必须早于 generated 和 minified-vendor

`packages/apps-kit/core/i18n/src/meta/{ssg}/{lang}.js` 是单行 minified JSON。如果按「内容优先」检测，会出现两类误判：

- **误判 minified-vendor**：`lines==1 && size>5000` 命中
- **误判 generated**：内容里有 SEO 文本「Auto-Generated Report」，命中正则
  **正确顺序**：i18n-data 路径检测必须在 generated / minified-vendor 之前。

### 2. generated 仅检测注释行

正则 `/auto[-\s]?generated|generated by/i` 单纯应用于首行会被 SEO 文本（"Auto-Generated Report"）、商业文案（"generated by AI"）误命中。**必须**约束为注释行：

```regex
/^\s*(?:\/\/|\/\*|\*)[^\n]{0,200}(?:code generated|@generated|auto[-\s]?generated|generated by)/i
```

真生成产物头部都是 `// code generated by xxx` 或 `/* @generated */`，加注释前缀就能精确识别。

### 3. logic-code 关键字检测必须严格

朴素正则 `\bclass\b` 会命中 HTML 模板里的 `class="..."`（如邮件模板含大量 `<div class="...">`）。必须严格化：

- `class` → `\bclass\s+\w+`（class 后必须跟标识符）
- `function` → `\bfunction[\s*(]`（function 后必须是空格、星号或括号）
- 模板字符串 `\`...\`` 内部含 JS 关键字也会误命中，string-template 检测时**先把模板字符串掏空**再判逻辑

### 4. .jsx 必须早于 barrel-only

3 行的 `index.jsx` 形如 `export { Foo } from './bar'` 会被 barrel-only 命中，但同目录已有 .tsx 兄弟时它应当一起转 .tsx 保持一致。**正确顺序**：jsx 扩展名检测优先于 barrel-only。

### 5. chart 适配器的误判风险

`packages/apps-kit/components/chart/k-chart/lib/trading-view/library/` 下的 .js 文件首行常常是 `/**` 注释，不是 import 行。**不能**用「首行含 `@/` import」作为 chart-adapter 触发条件。改为「路径在 `chart/k-chart/lib/`」即可，因为已知该目录下唯一的真 vendor 是 `charting_library.esm.js`，已通过文件名白名单（vendor-sdk）排除。

### 6. server.js 判定依赖项目状态

规则的 `next-server` 标签依赖「项目无 tsx/ts-node 依赖」这个事实。如果未来引入了 tsx，本规则需要更新。检测命令：

```bash
grep -E '"(tsx|ts-node)"\s*:' package.json apps/*/package.json packages/*/package.json | grep -v apps-icons
```

### 7. credential 必须最优先

含敏感凭证的脚本（如 `credentials.js`、`token.js`）路径包含 `/scripts/`，会先命中 node-script。**必须**让 credential 检测排在最前，确保安全问题不被掩盖。

### 8. 「建议转」≠「值得转」

skill 输出 `chart-adapter` / `logic-code` / `jsx-mixed` 表示**技术上能转**，但是否「值得」由人评估。例如 `save_load_adapter.js` 1717 行 95% 是 TradingView 配置数据，末尾几行才是方法——会被归到 chart-adapter 但实际收益低。报告应在「优先级建议」中标注此类文件让用户人工决定。

## 注意事项

- 不修改任何源文件
- 灰色地带不强行归类，留给人工 review
- 凭证文件标记为 `credential` 后**优先级最高**，即使路径在 `scripts/` 也以 credential 输出（让安全问题不被脚本类掩盖）
- 报告末尾给「优先级建议」：先做 jsx-mixed（一致性）、再做 chart-adapter（业务核心）、最后做 logic-code
