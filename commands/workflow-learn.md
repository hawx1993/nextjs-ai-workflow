# 触发

在解决了一个非简单（具有一定复杂度和价值）的问题后，可以在当前会话的任意时刻执行 `/byd-next:workflow-learn` 命令。

# 从当前会话提炼可复用经验

请使用 Skill 工具调用 `byd:workflow-learn`，并将 `$ARGUMENTS` 原样作为参数传入。

## 目标

分析当前 Claude Code 会话，识别对 BYDFi Web 仓库未来开发、审查、修复或工作流配置有复用价值的经验，并按项目规范沉淀为候选 skill、工作指导或可迁移规则。

## 适用场景

在以下场景后使用本命令：

- 解决了一个非显而易见的 Next.js / React / TypeScript / SSR / SSG 问题。
- 总结出可复用的 BYDFi 页面实现、组件拆分、i18n、theme、RTL 或响应式处理模式。
- 发现了某类构建、lint、hook、commit、PR、Figma、设计系统或 Claude Code workflow 的稳定处理方式。
- 多次遇到相同问题，希望转化为 `.claude/skills/*`、`.claude/rules/*` 或 command 文档中的明确约束。
- 需要把当前会话的经验整理成“以后遇到类似问题应该怎么做”。

不适合沉淀的内容：

- 简单拼写错误、一次性 typos、纯格式化改动。
- 已经由代码、README、git 历史或 `.claude/CLAUDE.md` 明确记录的普通事实。
- 只对本次需求有效、未来不应复用的临时方案。
- 依赖偶发外部状态的结论，例如临时网络故障或一次性接口异常。

## 参数

- `$ARGUMENTS` 为空：默认分析当前会话，列出可提炼模式，并在保存前请求用户确认。
- `$ARGUMENTS` 包含方向说明：按说明聚焦分析，例如：
  - `i18n`
  - `theme`
  - `commit workflow`
  - `ssr page pattern`
  - `review rule`
- `$ARGUMENTS` 包含 `--dry-run`：只输出候选经验和建议落点，不写文件。
- `$ARGUMENTS` 包含 `--save`：仍需先展示待保存内容并获得用户确认；不得静默写入。

## 必须遵守

- 必须先遵守 `.claude/CLAUDE.md` 和相关 `.claude/rules/*`。
- 必须优先复用现有 `.claude/skills/*`、`.claude/rules/*`、`.claude/commands/byd/*` 的结构和命名风格。
- 保存前必须检查是否已有相同主题的 skill、rule 或 command；有则建议更新已有文件，不要重复创建。
- 不得把敏感信息、账号、token、私有接口响应、用户隐私或一次性业务数据写入 skill。
- 不得把“本次做了什么”当作可复用经验；必须提炼为“未来遇到什么情况，应该如何处理”。
- 如果建议写入项目文件，必须说明：
  - 推荐文件路径
  - 提炼原因
  - 适用触发条件
  - 具体内容草案
  - 验证或维护方式
- 未经用户确认，不得写入或覆盖任何文件。

## 推荐沉淀位置

| 内容类型                          | 推荐位置                     | 说明                                              |
| --------------------------------- | ---------------------------- | ------------------------------------------------- |
| 可由 slash command 直接调用的流程 | `.claude/commands/byd/*.md`  | 入口命令，只负责路由、参数规则和执行要求          |
| 可复用的执行流程或专家能力        | `.claude/skills/**/SKILL.md` | 适合沉淀完整工作流、检查清单、工具使用顺序        |
| 全项目硬规则                      | `.claude/rules/*.md`         | 适合 i18n、theme、React、TypeScript、验证等强约束 |
| 个人偏好或项目外事实              | Claude memory                | 只保存代码仓库无法表达、未来仍有价值的信息        |
| 项目的文档或专业知识              | `.claude/docs/`              | 适合沉淀长期维护的项目知识、背景说明或参考资料    |

## 提炼维度

请重点寻找以下类型的可复用模式：

### 1. 错误处理模式

- 出现了什么错误？
- 根因是什么？
- 哪个最小改动修复了问题？
- 未来如何快速识别同类问题？

### 2. 调试与验证模式

- 哪些命令组合有效？
- 哪些检查顺序能减少误判？
- 是否需要补充到 `verification.md` 或某个 workflow skill？

### 3. BYDFi 项目约定

- 是否发现了新的页面、路由、布局、数据获取、SEO、i18n、theme 或 monorepo 约定？
- 是否需要补充到 `nextjs.md`、`react-tsx.md`、`language.md`、`theme.md` 或对应 skill？

### 4. 前端质量规则

- 是否发现可复用的 React / TSX 审查规则？
- 是否涉及未使用样式、未使用变量、硬编码颜色、手写断点、DOM style RTL 风险等？
- 是否应补充到 `dev/nextjs-review/SKILL.md`？

### 5. Git / PR / workflow 规则

- 是否发现 commit、push、PR、分支、hook 处理中的稳定流程？
- 是否应更新 `workflow-git-commit`、`workflow-create-pr` 或相关 skill？

## 输出格式

```text
结论：有可沉淀内容 / 暂无值得沉淀内容

候选经验：
- 名称：<经验名称>
  类型：skill / rule / command / memory / 不建议保存
  推荐位置：<路径>
  触发条件：<未来何时使用>
  提炼原因：<为什么值得复用>
  内容草案：<可直接写入或修改的摘要>

需要用户确认：
- 1. 保存到推荐位置
- 2. 修改草案后保存
- 3. 只输出，不保存
- 4. 取消
```

## 保存要求

如果用户确认保存：

1. 读取目标文件当前内容；目标文件不存在时说明将新建。
2. 只做与本次提炼相关的最小修改。
3. 保持中文说明、表格风格和现有 command / skill 结构一致。
4. 修改后说明改动文件和后续验证建议。
5. 如果改动 `.claude` 配置、command 或 skill，建议至少运行相关轻量检查；如无法运行，明确说明原因。

## 注意事项

- 本命令只负责学习和沉淀经验，不直接修改业务代码。
- 不要为了“学习”而扩大范围重构 `.claude` 目录。
- 不要生成过宽泛、空洞的经验，例如“写代码前先理解需求”。
- 每条经验应能在未来明确触发，并能指导具体动作。
- 对与现有规则冲突的经验，必须先指出冲突并请求用户确认，不得直接覆盖。
